React에는 Suspense와 lazy가 있고, Error Boundary를 구현할 수 있는 인터페이스를 제공합니다. 하지만 실제 애플리케이션에서 사용해 보면 빈틈이 드러납니다 — Suspense는 SSR에서 깨지고, Error Boundary는 직접 구현하려면 보일러플레이트가 많고, 이 모든 것을 조율할 방법이 없습니다. React뿐 아니라, 데이터 페칭 라이브러리와 상태 관리 라이브러리에도 각각의 빈틈이 있습니다.

**Suspensive는 그 빈틈을 채웁니다.** 비동기 렌더링, 에러 처리, 데이터 페칭이 더 적은 코드와 더 적은 우회로 제대로 동작하도록 선언적 컴포넌트와 훅을 제공합니다.

## 버전 지원 및 호환성

Suspensive는 다음과 같은 버전의 종속성을 지원합니다:

- React 18 및 React 19
- TanStack Query v4 및 v5
- Jotai v2

## 주요 이점

- **선언적 오류 처리**: try/catch 블록 대신 구성 요소 기반 오류 경계
- **간소화된 데이터 가져오기**: TanStack Query 사용 시 비동기 상태 통합
- **조정된 오류 관리**: 여러 오류 경계 조정
- **제어된 로딩 상태**: 타이밍 컨트롤을 통한 로딩 인디케이터 세밀 조정
- **뷰포트 인식 구성 요소**: 뷰포트 인식 구성 요소 만들기
