# Getting Started

Build your first async-safe React page with Suspensive in 5 minutes.

<Steps>

### Install packages

```shell npm2yarn
npm install @suspensive/react @suspensive/react-query @tanstack/react-query
```

### Wrap your app with ErrorBoundary and Suspense

Replace manual loading/error state checks with declarative boundaries:

```tsx
import { ErrorBoundary, Suspense } from '@suspensive/react'

function App() {
  return (
    <ErrorBoundary
      fallback={({ error, reset }) => (
        <div>
          <p>Something went wrong: {error.message}</p>
          <button onClick={reset}>Try again</button>
        </div>
      )}
    >
      <Suspense fallback={<div>Loading...</div>}>
        <UserProfile />
      </Suspense>
    </ErrorBoundary>
  )
}
```

### Fetch data with SuspenseQuery

Use `<SuspenseQuery/>` to render async data as JSX — no hooks, no wrapper components:

```tsx
import { SuspenseQuery } from '@suspensive/react-query'

function UserProfile() {
  return (
    <SuspenseQuery queryKey={['user']} queryFn={fetchUser}>
      {({ data: user }) => (
        <div>
          <h2>{user.name}</h2>
          <p>{user.email}</p>
        </div>
      )}
    </SuspenseQuery>
  )
}
```

### Add SSR safety with clientOnly

If you're using Next.js or another SSR framework, prevent hydration issues:

```tsx
import { Suspense } from '@suspensive/react'

function Page() {
  return (
    <Suspense fallback={<Skeleton />} clientOnly>
      <UserProfile />
    </Suspense>
  )
}
```

### Group error boundaries for coordinated resets

Reset multiple error boundaries at once without prop drilling:

```tsx
import { ErrorBoundary, ErrorBoundaryGroup } from '@suspensive/react'

function Dashboard() {
  return (
    <ErrorBoundaryGroup>
      <ErrorBoundaryGroup.Consumer>
        {({ reset }) => <button onClick={reset}>Reset All</button>}
      </ErrorBoundaryGroup.Consumer>
      <ErrorBoundary fallback={<UserError />}>
        <UserSection />
      </ErrorBoundary>
      <ErrorBoundary fallback={<StatsError />}>
        <StatsSection />
      </ErrorBoundary>
    </ErrorBoundaryGroup>
  )
}
```
