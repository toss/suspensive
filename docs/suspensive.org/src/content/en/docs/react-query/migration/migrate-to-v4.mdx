# Migrating to v4

## Handling BREAKING CHANGES

### `@suspensive/react-query` package is removed

<Callout type='info'>

This affects every user upgrading from v3 to v4.

</Callout>

The unified `@suspensive/react-query` package has been removed. Switch to the version-specific package that matches the major version of `@tanstack/react-query` you depend on:

- `@tanstack/react-query` v4 → `@suspensive/react-query-4`
- `@tanstack/react-query` v5 → `@suspensive/react-query-5`

### Backported APIs are removed from `@suspensive/react-query-{4,5}`

APIs that have been backported into `@tanstack/react-query` itself (e.g. `useSuspenseQuery`, `queryOptions`, `mutationOptions`, `usePrefetchInfiniteQuery`, and their option/result types) are no longer re-exported from `@suspensive/react-query-{4,5}`. Import them directly from `@tanstack/react-query`.

```diff
- import { useSuspenseQuery, SuspenseQuery, queryOptions } from '@suspensive/react-query'
+ import { useSuspenseQuery, queryOptions } from '@tanstack/react-query'
+ import { SuspenseQuery } from '@suspensive/react-query-5'
```

## Recommended migration path

Run the [`migrate-to-v4`](/docs/codemods/migrateToV4) codemod once to resolve both code-level breaking changes in a single pass:

```bash filename="Terminal"
npx @suspensive/codemods migrate-to-v4 .
```

Then update your `package.json` dependencies manually:

```diff
{
  "dependencies": {
-   "@suspensive/react-query": "^3.0.0",
+   "@suspensive/react-query-5": "^4.0.0",
    "@tanstack/react-query": "^5.0.0"
  }
}
```

<Callout type='info'>

`@suspensive/react` and `@suspensive/jotai` also bump to v4 alongside this release, but they have no code-level breaking changes — only a coordinated major version increment.

</Callout>
